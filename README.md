# 🧶 Научный клубок — карта знаний R&D для горно-металлургической отрасли

Единый верифицируемый граф знаний: публикации, эксперименты, материалы, процессы,
оборудование, эксперты. Запросы на естественном языке с числовыми ограничениями,
географией и временными диапазонами.

**Масштаб на полном корпусе кейса:** 1 828 уникальных документов (PDF/DOCX/PPTX/XLS, RU+EN, включая OCR сканов
на GPU; дубликаты корпуса схлопнуты по контент-хэшу), **259 304 уникальных числовых условия** (368 428 сырых, дубли схлопнуты
в occurrences; 100% верифицированы string-match), 33 861 упоминание сущностей, 666 LLM-утверждений (85 документов), 17 экспертов, 3 000 рёбер противоречий.
Ответ на сложный запрос — 2–3 с. Три LLM-провайдера с автопереключением:
YandexGPT → GigaChat → DeepSeek (open-weights через OpenRouter).

## Ключевая идея архитектуры

**Гибрид с жёстким разделением ответственности:**

| Слой | Кто отвечает | Почему |
|---|---|---|
| Числа, единицы, диапазоны | **Только детерминированная regex-грамматика** | ТЗ: «ошибки в извлечении концентраций недопустимы». LLM числа не генерирует вообще |
| Верификация | String-match каждой цитаты к исходному тексту | Для **чисел** галлюцинация исключена полностью; для утверждений обязательна дословная цитата-основание из источника (соответствие формулировки цитате — зона ответственности эксперта при ручной модерации) |
| Сущности и связи | LLM по строгой JSON-схеме онтологии + тезаурус-канонизатор | Высокий recall, RU/EN синонимы («электроэкстракция» ↔ «electrowinning» ↔ «ЭЭ») |
| Хранение | Kùzu — embedded графовая СУБД с Cypher | Обход 3–4 уровней связей < 0.2 с |
| Ответы | GraphRAG: граф выполняет фильтры детерминированно, LLM только оформляет | Каждое утверждение в ответе = узел графа с дословной цитатой и источником |

> Граф — источник истины. LLM — интерфейс к нему на входе и выходе, но никогда — источник фактов.

## Что умеет

- **Многопараметрические запросы со строгим AND**: «методы обессоливания при сульфатах
  200–300 мг/л и сухом остатке ≤1000 мг/дм³» → парсер извлекает сущности + операторы +
  диапазоны + единицы, а граф вычисляет **пересечение документов, удовлетворяющих всем
  условиям одновременно** (семантика вложенности: «≤5000» НЕ подтверждает
  требование «≤1000»). Пример: «электролиз никеля: 55–70 °С и 150–300 А/м²» →
  3 строгих документа из 1828; если пересечение пусто — система честно
  показывает это как пробел, а частичные совпадения — отдельно
- **Числовые ограничения**: `≤`, `≥`, `не более`, диапазоны «60–80 °С», «около 20 л/мин»;
  сопоставление интервалов запроса и корпуса с нормализацией единиц (мг/дм³ = мг/л)
- **География**: отечественная / зарубежная / сравнительный режим (фильтры применяются
  ко всем слоям ответа: публикации, утверждения, числовые условия)
- **Временные диапазоны**: «за последние 5 лет», «с 2018», «2010–2020»
- **Модель верификации**: каждое утверждение хранит в узле графа дословную цитату
  и флаг verified + источник + уровень достоверности (🟢🟡🔴) + дата актуализации
  + статус (active / unverified_quote / superseded с указанием заместившего факта)
- **Зоны разногласий**: автоматическое обнаружение непересекающихся числовых интервалов
  по одному (параметр, вещество) из разных источников — 3000 рёбер CONTRADICTS в графе
- **Пробелы в знаниях**: комбинации «материал–режим–условие» без покрытия в корпусе
- **Мониторинг**: поиск новых внешних публикаций по теме (Yandex Search API)
- **Дашборд**: покрытие по доменам (гидро/пирометаллургия, экология...), гео-разрез

## Качество и безопасность

- **Дедупликация корпуса** по контент-хэшу: 2090 сырых файлов → 1828 уникальных
  документов (дубли из вложенных папок архива схлопнуты, статистика честная)
- **Детерминированность**: одинаковый запрос → одинаковый набор фактов
  (упорядоченные выборки, стабильные ID сущностей и экспертов)
- **Golden-тесты**: `scripts/test_20_questions.py` — 20 типов запросов из ТЗ,
  для ключевых проверяются ожидаемые числа в ответе (60–65°C, 240 А/м², 98%...)
- **Защита веб-слоя**: санитизация всего рендеримого контента (DOMPurify),
  `/api/audit` только по админ-токену, безопасная распаковка бандла
  (защита от path traversal, опциональная проверка SHA256)

## Архитектура

```
документы (PDF/DOCX, RU/EN)
   │  ingest.py (PyMuPDF, python-docx, langdetect)
   ▼
нормализованный текст + метаданные (год, язык, гео)
   │
   ├── numeric.py      — regex-грамматика: 259 000+ числовых условий, 100% string-match
   ├── thesaurus.py    — канонизатор сущностей RU/EN (88 канонов, 400+ синонимов)
   └── LLM-экстракция  — claims/relations/experts по строгой JSON-схеме
                          (все цитаты повторно верифицируются string-match)
   ▼
Kùzu (Cypher): Material, Process, Equipment, Facility, Publication,
               Condition, Claim, Expert + 19 типов рёбер
   ▲
   │ детерминированные фильтры (числа, гео, годы) + BM25 + сущности
search.py ◄── parse_query (та же грамматика чисел, что и для корпуса!)
   │
answer.py — структурированный литобзор: выводы с достоверностью, таблица чисел
            с цитатами, источники, разногласия, пробелы
   │
FastAPI + cytoscape.js — чат, живой граф, фильтры, карточки узлов
```

## Запуск

```bash
pip install -r requirements-core.txt   # ядро; для OCR сканов см. requirements.txt (GPU-torch)
# 1) распаковать корпус кейса (4.7GB zip, вкл. вложенные архивы):
python scripts/extract_corpus.py       # -> data/raw/full/
# 2) ингест всех форматов (pdf/docx/pptx/xls/xlsx/doc):
python backend/ingest_full.py          # -> data/cache/docs.jsonl + txt/
python backend/ingest_ocr.py           # OCR сканов на GPU (easyocr, опционально)
# 3) сборка графа (параллельно, полный текст, дедуп вместо потерь):
python backend/build_graph_mp.py
python backend/merge_extracted.py      # LLM-слой: claims/relations/experts
# 4) проверка целостности и запуск:
python scripts/verify_graph.py         # 6 проверок: полнота, рёбра, string-match...
python -m uvicorn backend.app:app --port 8017
# открыть http://127.0.0.1:8017
```

`.env` (не в репозитории):
```
YC_API_KEY=...            # ключ Yandex AI Studio (кейс)
YC_FOLDER_ID=...
OPENROUTER_API_KEY=...    # fallback-провайдер (опционально)
GIGACHAT_AUTH_KEY=...     # fallback-провайдер (опционально, нужен certs/russiantrustedca.pem)
LLM_ORDER=yandex,openrouter,gigachat   # порядок перебора провайдеров
```

При недоступности LLM система работает в граф-режиме: детерминированный
поиск + шаблонный синтез (все факты сохраняются, теряется только «полировка» текста).

## Масштабируемость

- Новые домены = новые записи тезауруса + прогон пайплайна (онтология не меняется)
- Kùzu держит миллионы узлов на одной машине; переход на Neo4j — те же Cypher-запросы
- Извлечение инкрементально: новый документ проходит слои независимо
- Версионирование фактов: Claim.status / superseded_by / created_at
