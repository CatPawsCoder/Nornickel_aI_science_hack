# 🧶 Научный клубок — карта знаний R&D для горно-металлургической отрасли

Единый верифицируемый граф знаний: публикации, эксперименты, материалы, процессы,
оборудование, эксперты. Запросы на естественном языке с числовыми ограничениями,
географией и временными диапазонами.

**Масштаб на полном корпусе кейса:** 2 090 документов (PDF/DOCX/PPTX/XLS, RU+EN,
включая OCR сканов на GPU), **181 047 числовых условий** (100% верифицированы
string-match), 36 858 упоминаний сущностей, 233 LLM-утверждения, 27 экспертов.
Ответ на сложный запрос — 2–3 с. Три LLM-провайдера с автопереключением:
YandexGPT → OpenRouter → GigaChat.

## Ключевая идея архитектуры

**Гибрид с жёстким разделением ответственности:**

| Слой | Кто отвечает | Почему |
|---|---|---|
| Числа, единицы, диапазоны | **Только детерминированная regex-грамматика** | ТЗ: «ошибки в извлечении концентраций недопустимы». LLM числа не генерирует вообще |
| Верификация | String-match каждой цитаты к исходному тексту | Галлюцинация физически не может попасть в граф |
| Сущности и связи | LLM по строгой JSON-схеме онтологии + тезаурус-канонизатор | Высокий recall, RU/EN синонимы («электроэкстракция» ↔ «electrowinning» ↔ «ЭЭ») |
| Хранение | Kùzu — embedded графовая СУБД с Cypher | Обход 3–4 уровней связей < 0.2 с |
| Ответы | GraphRAG: граф выполняет фильтры детерминированно, LLM только оформляет | Каждое утверждение в ответе = узел графа с дословной цитатой и источником |

> Граф — источник истины. LLM — интерфейс к нему на входе и выходе, но никогда — источник фактов.

## Что умеет

- **Многопараметрические запросы**: «методы обессоливания при сульфатах 200–300 мг/л
  и сухом остатке ≤1000 мг/дм³» → парсер извлекает сущности + операторы + диапазоны + единицы
- **Числовые ограничения**: `≤`, `≥`, `не более`, диапазоны «60–80 °С», «около 20 л/мин»;
  сопоставление интервалов запроса и корпуса с нормализацией единиц (мг/дм³ = мг/л)
- **География**: отечественная / зарубежная / сравнительный режим
- **Временные диапазоны**: «за последние 5 лет», «с 2018», «2010–2020»
- **Модель верификации**: источник + уровень достоверности (🟢🟡🔴) + дата актуализации
  + статус (active / unverified_quote / superseded)
- **Зоны разногласий**: автоматическое обнаружение непересекающихся числовых интервалов
  по одному параметру из разных источников
- **Пробелы в знаниях**: комбинации «материал–режим–условие» без покрытия в корпусе
- **Мониторинг**: поиск новых внешних публикаций по теме (Yandex Search API)
- **Дашборд**: покрытие по доменам (гидро/пирометаллургия, экология...), гео-разрез

## Архитектура

```
документы (PDF/DOCX, RU/EN)
   │  ingest.py (PyMuPDF, python-docx, langdetect)
   ▼
нормализованный текст + метаданные (год, язык, гео)
   │
   ├── numeric.py      — regex-грамматика: 181 000+ числовых условий, 100% string-match
   ├── thesaurus.py    — канонизатор сущностей RU/EN (88 канонов, 400+ синонимов)
   └── LLM-экстракция  — claims/relations/experts по строгой JSON-схеме
                          (все цитаты повторно верифицируются string-match)
   ▼
Kùzu (Cypher): Material, Process, Equipment, Facility, Publication,
               Condition, Claim, Expert + 19 типов рёбер
   ▲
   │ детерминированные фильтры (числа, гео, годы) + BM25 + сущности
search.py ◄── parse_query (та же грамматика чисел, что и для корпуса!)
   │
answer.py — структурированный литобзор: выводы с достоверностью, таблица чисел
            с цитатами, источники, разногласия, пробелы
   │
FastAPI + cytoscape.js — чат, живой граф, фильтры, карточки узлов
```

## Запуск

```bash
pip install -r requirements.txt
# положить документы в data/raw/obzory/, затем:
python backend/ingest.py          # текст + метаданные
python -c "import sys; sys.path.insert(0,'backend'); from numeric import *; ..."  # см. scripts/
python backend/build_graph.py     # граф: сущности, упоминания, условия
python backend/merge_extracted.py # LLM-слой: claims/relations/experts
python -m uvicorn backend.app:app --port 8017
# открыть http://127.0.0.1:8017
```

`.env` (не в репозитории):
```
YC_API_KEY=...      # ключ Yandex AI Studio
YC_FOLDER_ID=...
```

При недоступности LLM система работает в граф-режиме: детерминированный
поиск + шаблонный синтез (все факты сохраняются, теряется только «полировка» текста).

## Масштабируемость

- Новые домены = новые записи тезауруса + прогон пайплайна (онтология не меняется)
- Kùzu держит миллионы узлов на одной машине; переход на Neo4j — те же Cypher-запросы
- Извлечение инкрементально: новый документ проходит слои независимо
- Версионирование фактов: Claim.status / superseded_by / created_at

## Соответствие FAIR

- **Findable**: BM25 + сущности + фильтры; каждый факт имеет ID узла
- **Accessible**: REST API, экспорт JSON; роли (см. roadmap)
- **Interoperable**: онтология сериализуема в RDF/OWL; canon ID = URI-ready
- **Reusable**: каждый факт несёт источник, цитату, достоверность, дату
